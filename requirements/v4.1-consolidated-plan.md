### Task 0.7: Separate Health Insurance from Business Insurance
- **What**: Both Hiscox (business liability) and health insurance premiums are categorized as `INSURANCE`
- **Problem**: Business insurance → Schedule C Line 15. Self-employed health insurance → Form 1040 Line 17 (adjustment to income). These are different tax treatments. Using a subcategory does NOT work — both the IRS line mapping and FreeTaxUSA export route by `tax_category`, not `tax_subcategory`.
- **Fix — MUST be done in this order**:
  1. Add `HEALTH_INSURANCE = "HEALTH_INSURANCE"` to `TaxCategory` in `src/models/enums.py`
  2. Add `"HEALTH_INSURANCE"` to `IRS_LINE_MAPPING` in `tax_export.py` mapped to `"1040 Line 17 — Self-employed health insurance deduction"`
  3. Add `HEALTH_INSURANCE` entry to `SCHEDULE_C_LINES` in `freetaxusa.py` (or a new `FORM_1040_LINES` mapping since it's NOT a Schedule C item)
  4. Add `<option>` for `HEALTH_INSURANCE` to `TransactionCard.svelte` category dropdown
  5. Add `HEALTH_INSURANCE` to `categories.ts` in the dashboard
  6. Run `pytest` to verify no regressions
  7. **THEN** PATCH affected health insurance transactions via API to use the new category
- **Why ordering matters**: If you PATCH before adding the enum, the API returns 422 validation errors. If you add the enum but skip the export mapping, health insurance silently drops from tax exports.
- **Effort**: 45-60 minutes (enum + 3 mapping files + frontend + PATCH + tests)

### Task 0.8: Verify Sign Convention System-Wide
